# Интеграция локальной модели Ollama

## Описание
Реализована поддержка локальной модели llama3.2:3b через Ollama в дополнение к существующей интеграции с OpenRouter. Локальная модель работает без API ключей, но имеет ограничения по функциональности.

## Установка и настройка Ollama

### 1. Установка Ollama
```bash
# Linux
curl -fsSL https://ollama.com/install.sh | sh

# macOS
brew install ollama

# Windows
Скачать с https://ollama.com/download
```

### 2. Запуск сервера Ollama
```bash
# Запуск сервера в фоновом режиме
ollama serve &

# Или запуск как демон
sudo systemctl start ollama
```

### 3. Загрузка модели
```bash
# Загрузить модель llama3.2:3b (≈1.8GB)
ollama pull llama3.2:3b

# Проверить установленные модели
ollama list
```

## Настройка приложения

### 1. Обновление .env файла
Добавьте или убедитесь, что есть следующие строки в `.env`:
```env
# Ollama (локальная модель)
OLLAMA_URL=http://localhost:11434
OLLAMA_MODEL=llama3.2:3b

# OpenRouter API (для облачного режима)
OPENROUTER_API_KEY=your_api_key_here
```

### 2. Запуск приложения
```bash
# Запуск enhanced версии с поддержкой локальной модели
python3 run_app.py --enhanced
```

## Использование в интерфейсе

### Выбор модели:
1. Откройте веб-интерфейс (обычно http://localhost:7860)
2. В боковой панели найдите секцию "Модель LLM"
3. Выберите режим:
   - **Чекбокс "Использовать локальную модель Ollama"** - для локальной модели
   - **Dropdown "Модель OpenRouter"** - для облачной модели (только если не выбрана локальная)
4. Нажмите "Инициализировать модель"

### Ограничения локальной модели:
- ❌ **Недоступны MCP инструменты** (расчет маршрутов, стоимости доставки)
- ✅ **Доступен RAG** (поиск в документах по тарифам, правилам)
- ✅ **Доступны базовые знания** (ответы на общие вопросы)

## Архитектура

### Компоненты:
1. **LLMClient** - абстрактный класс для работы с LLM
   - `OllamaClient` - для локальной модели
   - `OpenRouterClient` - для облачной модели
   
2. **EnhancedRoutePlannerAgent** - поддерживает оба режима
   - `use_local` параметр для выбора типа модели
   - Автоматическое переключение между промптами

3. **UI с Gradio** - интерфейс для выбора модели

### Поток данных:
```
Пользователь → UI (выбор модели) → EnhancedRoutePlannerAgent → LLMClient → Модель
```

## Устранение неполадок

### Проблема: "Не удалось подключиться к серверу Ollama"
**Решение:**
```bash
# Проверить статус сервера
curl http://localhost:11434/api/tags

# Перезапустить сервер
ollama serve
```

### Проблема: Модель не загружается
**Решение:**
```bash
# Проверить установленные модели
ollama list

# Перезагрузить модель
ollama pull llama3.2:3b
```

### Проблема: RAG не работает
**Решение:**
- Убедитесь, что файл `metadata.db` существует
- Проверьте логи при запуске приложения

## Производительность

### Ожидаемые характеристики:
- **Локальная модель**: быстрое время отклика, не требует интернета
- **OpenRouter модель**: более точные ответы, поддержка инструментов
- **Память**: ~2GB RAM для llama3.2:3b
- **Скорость**: 5-10 токенов/сек на CPU

## Безопасность

### Преимущества локальной модели:
- 🔒 Данные не покидают вашу инфраструктуру
- 🔒 Не требуются API ключи
- 🔒 Работает без интернет-соединения

### Рекомендации:
1. Используйте локальную модель для конфиденциальных данных
2. Используйте OpenRouter для сложных расчетов
3. Регулярно обновляйте модель Ollama
