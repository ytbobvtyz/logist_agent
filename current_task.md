## Задача Day 20: Добавление MCP сервера ПЭК для расчёта стоимости доставки

### Цель
Добавить второй MCP сервер для интеграции с API ПЭК (ПЭК) для расчёта стоимости доставки грузов. Агент должен уметь рассчитывать стоимость для каждого сегмента маршрута, используя данные из Яндекс API.

### Требования

#### 1. Создать файл `mcp_servers/pecom_server.py`

MCP сервер с инструментом `calculate_cost`:

**Входные параметры:**
- `from_city: str` — город отправления
- `to_city: str` — город назначения
- `weight_kg: float` — вес в кг (по умолчанию 50)
- `length_m: float` — длина в метрах (по умолчанию 0.5)
- `width_m: float` — ширина в метрах (по умолчанию 0.5)
- `height_m: float` — высота в метрах (по умолчанию 0.4)

**Выходные данные (JSON):**
- `success: bool`
- `from_city: str`
- `to_city: str`
- `weight_kg: float`
- `cost: int` (стоимость в рублях)
- `message: str` (читаемое сообщение)

3. Формула запроса к API ПЭК
API ПЭК (публичный, без ключа):

URL: http://calc.pecom.ru/bitrix/components/pecom/calc/ajax.php

Метод: GET

Параметры:

take[town] — ID города отправления

deliver[town] — ID города назначения

places[0][] — массив из 7 значений:

[длина, ширина, высота, объём, вес, негабарит, упаковка]

негабарит всегда 0, упаковка всегда 0

Объём рассчитывается: length_m * width_m * height_m

Пример запроса:

text
http://calc.pecom.ru/bitrix/components/pecom/calc/ajax.php?take[town]=446&deliver[town]=64883&places[0][]=0.5&places[0][]=0.5&places[0][]=0.4&places[0][]=0.1&places[0][]=50&places[0][]=0&places[0][]=0
