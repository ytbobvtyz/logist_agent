# Pecom API Cities Fetcher

## Описание
Простой скрипт для получения всех доступных городов из публичного API ПЭК.

## Файлы
- `test_api_pecom.py` - Основной скрипт для получения городов
- `sities_pec.json` - Файл с данными городов (создается автоматически)

## Использование

### 1. Запуск скрипта
```bash
python test_api_pecom.py
```

### 2. Что делает скрипт:
1. Подключается к публичному API ПЭК: `https://pecom.ru/ru/calc/towns.php`
2. Получает список всех доступных городов в формате JSON
3. Сохраняет данные в файл `sities_pec.json`
4. Выводит статистику по регионам и городам

### 3. Формат данных в `sities_pec.json`:
```json
{
    "Название региона1": {
        "ID_города1": "Название города1",
        "ID_города2": "Название города2",
        ...
    },
    "Название региона2": {
        "ID_города3": "Название города3",
        ...
    }
}
```

### 4. Пример данных:
```json
{
    "Абакан": {
        "-584988": "Абакан",
        "587947": "Абаза",
        "587961": "Аскиз (Аскизский р-н)"
    },
    "Москва": {
        "446": "Москва",
        "-290064": "Москва Центр",
        "-193819": "Аэропорт Домодедово"
    }
}
```

### 5. Требования
- Python 3.6+
- Библиотека `requests` (устанавливается через pip)

### 6. Установка зависимостей (если нужно)
```bash
pip install requests
```

### 7. Интеграция с существующим проектом
Существующий проект уже имеет модуль для работы с API ПЭК:
- `route_planner/pecom_server.py` - MCP сервер для расчетов стоимости доставки
- `data/carriers/pecom_api_doc.txt` - Документация API ПЭК

Полученные ID городов можно использовать для обновления словаря `PECOM_CITY_IDS` в `pecom_server.py`:
```python
PECOM_CITY_IDS = {
    "москва": 446,
    "санкт-петербург": -203,
    # ... дополнительные города из sities_pec.json
}
```

### 8. Запуск теста
Убедитесь, что скрипт работает:
```bash
python test_api_pecom.py
```

Ожидаемый вывод:

## Примечания
- API является публичным и не требует авторизации
- Данные обновляются в реальном времени с серверов ПЭК
- Скрипт обрабатывает ошибки подключения и парсинга
- Файл сохраняется в кодировке UTF-8 для поддержки кириллицы

## Использование данных
Полученные данные можно использовать для:
1. Поиска ID городов для API запросов
2. Автодополнения в формах ввода городов
3. Валидации вводимых пользователем городов
4. Обновления списка городов в существующем проекте
