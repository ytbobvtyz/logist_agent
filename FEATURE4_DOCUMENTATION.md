# 📋 FEATURE 4: Интеграция RAG с MCP

## 🎯 Цель
Реализовать интеллектуальную маршрутизацию между RAG и MCP инструментами, чтобы агент сам определял, какой инструмент использовать для конкретного запроса.

## 📊 Реализованные компоненты

### 1. Модификация `route_planner/agent.py`

#### Добавлена поддержка RAG:
- **Интеграция RAGRetriever**: Автоматическая загрузка RAG поиска при инициализации агента
- **Метод `search_with_rag()`**: Поиск релевантных чанков в документах
- **Метод `_build_rag_prompt()`**: Формирование промптов с контекстом из найденных документов
- **Метод `_should_use_rag()`**: Интеллектуальная логика выбора инструментов

#### Интеллектуальная маршрутизация:
- **Логика выбора инструментов**:
  - **RAG**: Для информационных запросов (тарифы, правила, обязанности, API документация)
  - **MCP**: Для расчетных запросов (расстояния, маршруты, стоимость доставки)
  - **Знания LLM**: Для общих вопросов

#### Обновленный SYSTEM_PROMPT:
```python
SYSTEM_PROMPT = """Ты умный логист-ассистент. У тебя есть инструменты:

## MCP ИНСТРУМЕНТЫ (для расчётов):
- mcp_geocode_batch - координаты городов
- mcp_find_optimal_route - оптимальный маршрут
- mcp_calculate_cost - стоимость доставки (через API ПЭК)

## RAG (для поиска в документах):
- Ты можешь искать информацию в документах (тарифы, правила, постановления)

## ПРАВИЛА ВЫБОРА ИНСТРУМЕНТА:
- Если вопрос про **расстояние, маршрут, стоимость доставки, координаты** → используй MCP
- Если вопрос про **правила, обязанности, тарифы из документов, API ПЭК** → сначала поищи в RAG
- Если вопрос про **общие понятия** → отвечай из своих знаний

"""
```

## 🚀 Ключевые особенности

### 1. Интеллектуальная маршрутизация
Агент анализирует запрос и автоматически выбирает подходящий инструмент:
- **Ключевые слова для RAG**: `тариф`, `правила`, `обязанности`, `постановление`, `закон`, `стоимость`, `цена`, `расценки`, `условия`, `требования`, `api`, `документ`, `инструкция`, `руководство`, `справочник`, `фрахтователь`, `фрахтовщик`, `перевозчик`, `экспедитор`
- **Ключевые слова для MCP**: `расстояние`, `маршрут`, `координаты`, `город`, `городa`, `рассчитать`, `стоимость доставки`, `цена перевозки`, `оптимальный путь`, `кратчайший маршрут`, `геокодирование`

### 2. Гибридный подход
- **Приоритет RAG для информационных запросов**: Сначала поиск в документах
- **Автоматическое переключение**: Если RAG не нашел информацию, используется LLM
- **Прозрачность работы**: Указание используемого инструмента в ответе

### 3. Обработка ошибок
- **Грациозная деградация**: При недоступности RAG или MCP система продолжает работать
- **Информативные сообщения**: Пользователь видит, какой инструмент использовался
- **Автоматическое восстановление**: Переподключение при сбоях

### Новые методы в `RoutePlannerAgent`:
1. **`_init_rag_retriever()`** - Инициализация RAG поиска
2. **`search_with_rag(query, top_k=3)`** - Поиск в документах
3. **`_build_rag_prompt(query, chunks)`** - Формирование промпта с контекстом
4. **`_should_use_rag(user_message)`** - Логика выбора инструмента
5. **`_process_with_mcp(user_message, callback)`** - Обработка с MCP инструментами
