# 🧪 Тестирование RAG с DeepSeek v3.2

## Обзор

`test_llm_rag.py` - специализированный тестовый скрипт для оценки эффективности RAG системы с использованием модели DeepSeek v3.2 через OpenRouter API.

## 📋 Функциональность

### Основные возможности:
- **Интеграция с DeepSeek v3.2** - использование платной модели для точного тестирования
- **Комплексное сравнение** - оценка эффективности RAG vs без RAG
- **Детальная аналитика** - метрики точности, времени ответа, качества ответов
- **Категориальный анализ** - группировка вопросов по темам
- **Автоматическое сохранение** - результаты в формате JSON

### Тестовые сценарии:
- 10 вопросов по логистике сгруппированы по категориям:
  - **Стоимость** (вопросы 1, 3, 4)
  - **Ограничения** (вопрос 2) 
  - **Техническое** (вопросы 5, 6, 7)
  - **Юридическое** (вопросы 8, 9, 10)

## 🚀 Использование

### Базовый запуск:
```bash
python test_llm_rag.py
```

### Быстрый тест (3 вопроса):
```bash
python test_llm_rag.py --quick
```

### Требования:
- Файл `.env` с `OPENROUTER_API_KEY`
- Установленные зависимости: `openai`, `python-dotenv`
- Доступ к модели DeepSeek v3.2 через OpenRouter

## 📊 Метрики оценки

### 1. **Точность ответов**
- Оценка по ключевым словам
- Процентное соотношение найденных/отсутствующих ключевых слов
- Сравнение точности с RAG и без RAG

### 2. **Производительность**
- Время ответа для каждого режима
- Анализ влияния RAG на скорость
- Сравнение среднего времени ответа
- **Процентное изменение времени** - ускорение/замедление в %
- **Анализ эффективности RAG** - 4 типа сценариев:
  - RAG быстрее и точнее ✅
  - RAG точнее, но медленнее ⚖️
  - RAG быстрее, но менее точен ⚡
  - RAG медленнее и менее точен ❌

### 3. **Качество ответов**
- Длина и структурированность ответов
- Наличие указаний на источники
- Детализация информации
- **Корневое сопоставление ключевых слов** - поиск по корням слов (фрахтователь → фрахт)

### 4. **Релевантность источников**
- Проверка корректности указанных источников
- Анализ использования документов
- Оценка релевантности найденных чанков

## 🆕 Новые возможности (обновление)

### 1. **Сравнение времени ответа**
- Добавлено процентное изменение времени ответа для каждого вопроса
- Интегрировано в итоговую таблицу результатов
- Расчет производительности RAG в процентах

### 2. **Анализ эффективности RAG**
- **4 типа эффективности** с умными выводами:
  - ✅ **RAG быстрее и точнее** - оптимальная эффективность
  - ⚖️ **RAG точнее, но медленнее** - решение о приоритетах
  - ⚡ **RAG быстрее, но менее точен** - скорость важнее точности
  - ❌ **RAG медленнее и менее точен** - требуется оптимизация

### 3. **Улучшенное сопоставление ключевых слов**
- **Корневое сопоставление** вместо точного совпадения
- Пример: "фрахтователь" → "фрахт", "перевозчик" → "перевоз"
- Повышение точности оценки ответов
