## 🚀 Быстрый старт

### Установка
```bash
# Клонирование репозитория
git clone <репозиторий>
cd logsit_agent

# Установка зависимостей
pip install -r requirements.txt

# Настройка API ключей
cp .env.example .env
# Отредактируйте .env файл, добавив ваш API ключ OpenRouter
```

### Запуск
```bash
python app/main.py
```

Откройте браузер и перейдите по адресу: http://localhost:7872

## 🎨 Интерфейс

### Общий вид
```
+----------------------------------------------------------+
| 🗺️ Logsit Agent - Умный логистический ассистент         |
+--------------------------+-------------------------------+
| ⚙️ Настройки            |                               |
|   [Выбор модели]        | 💬 Чат с ассистентом          |
|                         |                               |
| 💬 Управление диалогами | [История сообщений]           |
|   [Информация о диалоге]|                               |

|   [Переключиться] [Новый]|                               |
|                         |                               |
| 📡 Статус инструментов  |                               |
|   [Статус MCP и RAG]    |                               |
|   [Переподключить MCP]  |                               |
+--------------------------+-------------------------------+
```

### Элементы интерфейса

#### Левая панель (настройки)
1. **Выбор модели** - выбор LLM модели для ассистента
2. **Информация о диалоге** - статистика текущего диалога
3. **Список диалогов** - все доступные диалоги с индикаторами активности
4. **Статус MCP и RAG** - доступность инструментов расчета и поиска
5. **Отладка MCP** - лог вызовов инструментов

#### Правая панель (чат)
1. **История сообщений** - диалог между пользователем и ассистентом
2. **Поле ввода** - ввод сообщений пользователя
3. **Кнопка "Отправить"** - отправка сообщения

## 💬 Работа с диалогами

### Создание диалога
1. Нажмите кнопку **"➕ Новый диалог"**
2. Введите название диалога (или оставьте автоматическое)
3. Начните общение в новом контексте

### Переключение между диалогами
1. Выберите диалог из выпадающего списка
2. Нажмите кнопку **"↻ Переключиться"**
3. История чата обновится для выбранного диалога

### Управление диалогами
- **Очистка диалога** - удаляет все сообщения текущего диалога
- **Удаление диалога** - полностью удаляет диалог и все его данные
- **Автосуммаризация** - автоматически создается каждые 10 сообщений пользователя

### Статистика диалога
- **Количество сообщений** - общее число сообщений в диалоге
- **Сообщения пользователя** - количество сообщений от пользователя
- **Последняя суммаризация** - краткая сводка диалога
- **Статус суммаризации** - показывает, требуется ли обновление

## 🔧 Функции ассистента

### Интеллектуальный выбор инструментов
Ассистент автоматически определяет, какой инструмент использовать:

| Тип запроса | Инструмент | Иконка | Описание |
|------------|------------|--------|----------|
| Маршруты, расстояния, координаты | MCP | 🔧 | Расчеты через Model Context Protocol |
| Тарифы, правила, документы | RAG | 📚 | Поиск в документах перевозчиков |
| Общие вопросы | Знания | 💡 | Ответы из знаний LLM |

### MCP инструменты
- **geocode_batch** - получение координат городов
- **find_optimal_route** - поиск оптимального маршрута
- **calculate_cost** - расчет стоимости доставки

### RAG поиск
- Поиск в документах ПЭК, СДЭК, Почты России
- Семантический поиск с векторизацией
- Ранжирование результатов по релевантности

### Память задачи (Task State)
Ассистент отслеживает:
- **Цель диалога** - основная задача пользователя
- **Уточненные детали** - города, вес, сроки и т.д.
- **Ограничения** - бюджет, максимальное время, лимиты

## 📝 Примеры использования

### Пример 1: Расчет маршрута
```
Пользователь: Найди оптимальный маршрут между Москвой, Санкт-Петербургом и Казанью

Ассистент (🔧 MCP):
🗺️ Оптимальный маршрут:

Москва → Казань → Санкт-Петербург

📏 Расстояния:
• Москва → Казань: 820 км
• Казань → Санкт-Петербург: 1500 км

📊 Общее расстояние: 2320 км
```

### Пример 2: Поиск в документах
```
Пользователь: Какие тарифы на доставку до 50 кг?

Ассистент (📚 RAG):
🔍 Найдено в документах:

1. 📄 pecom.txt (схожесть: 92%)
   Тарифы на доставку до 50 кг: от 500 руб по Москве, от 1500 руб между регионами...

2. 📄 cdek.txt (схожесть: 85%)
   Тарифная сетка для грузов до 50 кг...
```

### Пример 3: Многошаговая задача
```
Пользователь: Нужно отправить груз 100 кг из Москвы в Сочи
[Ассистент запоминает: вес 100 кг, города Москва-Сочи]

Пользователь: Срок доставки 3 дня
[Ассистент добавляет: срок 3 дня]

Пользователь: Какая стоимость?
Ассистент (🔧 MCP):
📊 Расчет стоимости доставки 100 кг из Москвы в Сочи за 3 дня:

• Расстояние: 1620 км
• Тариф: 50 руб/км
• Итого: 81 000 руб
• С учетом срока: +10% = 89 100 руб
```

## 🛠️ Решение проблем

### Ошибка "Агент не инициализирован"
1. Проверьте наличие файла `.env` с OPENROUTER_API_KEY
2. Убедитесь, что интернет-соединение работает
3. Перезапустите приложение

### MCP инструменты не работают
1. Проверьте статус MCP в левой панели
2. Нажмите "🔄 Переподключить MCP"
3. Убедитесь, что скрипты MCP серверов доступны

### RAG поиск не работает
1. Проверьте статус RAG в левой панели
2. Убедитесь, что файлы индекса существуют:
   - `faiss_index` - векторный индекс
   - `metadata.db` - база метаданных
3. При необходимости запустите индексацию: `python indexer.py`

### Медленная работа
1. Проверьте размер базы данных `conversations.db`
2. Используйте более легковесную модель LLM
3. Ограничьте количество хранимых диалогов

### Ошибки LLM
1. Проверьте баланс OpenRouter аккаунта
2. Попробуйте бесплатную модель
3. Убедитесь, что API ключ валиден

## ❓ FAQ

### Q: Как сохраняются диалоги?
**A:** Все диалоги сохраняются в SQLite базу данных `conversations.db`. Сохраняются сообщения, суммаризации и состояние задачи.

### Q: Можно ли экспортировать диалоги?
**A:** В текущей версии экспорт не реализован. Диалоги хранятся только в локальной базе данных.

### Q: Сколько диалогов можно создать?
**A:** Теоретически неограниченно, но рекомендуется не более 100 для оптимальной производительности.

### Q: Как работает автосуммаризация?
**A:** Каждые 10 сообщений пользователя система создает краткую сводку диалога через LLM.

### Q: Можно ли отключить автосуммаризацию?
**A:** В текущей версии автосуммаризация всегда включена.

### Q: Какие модели LLM поддерживаются?
**A:** Все модели OpenRouter API. Рекомендуемые: openrouter/free (бесплатно), qwen/qwen3.6-plus-preview:free.

### Q: Нужен ли API ключ Яндекс.Карт?
**A:** Нет, это опционально. Без него будут использоваться приблизительные расчеты расстояний.

### Q: Как добавить свои документы для RAG поиска?
**A:** Положите текстовые файлы в папку `data/carriers/` и запустите `python indexer.py`.

### Q: Можно ли использовать с мобильных устройств?
**A:** Да, интерфейс адаптивен и работает на мобильных устройствах.

### Q: Как остановить приложение?
**A:** Нажмите Ctrl+C в терминале, где запущено приложение.
