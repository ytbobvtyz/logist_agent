# Архитектура Logsit Agent

## 🏗️ Общий обзор

Logsit Agent - это модульное веб-приложение для логистических расчетов с поддержкой множества диалогов, автосуммаризации и интеллектуального выбора инструментов.

## 📦 Модульная структура

### Уровень приложения (`app/`)

#### Компоненты UI (`app/components/`)
- **`sidebar.py`** - боковая панель с настройками и управлением диалогами
- **`chat.py`** - компонент чата для общения с ассистентом

#### Обработчики событий (`app/handlers/`)
- **`message_handler.py`** - обработка сообщений пользователя
- **`conversation_handler.py`** - управление операциями с диалогами

#### Главное приложение (`app/main.py`)
- Координация всех компонентов
- Регистрация обработчиков событий
- Управление жизненным циклом приложения

### Уровень бизнес-логики (`core/`)

#### Основной агент (`core/agent.py`)
- **`LogsitAgent`** - главный класс, координирующий все функции
- **`AgentConfig`** - конфигурация агента
- **`AgentState`** - состояние агента в runtime

#### Управление диалогами (`core/conversation_manager.py`)
- **`ConversationManager`** - управление множеством диалогов
- **`Conversation`** - модель диалога
- **`Message`** - модель сообщения
- **`TaskState`** - модель состояния задачи

#### Автосуммаризация (`core/summarizer.py`)
- **`Summarizer`** - генерация суммаризаций диалогов
- **`SummarizerConfig`** - конфигурация суммаризатора
- Триггер каждые 10 сообщений пользователя

#### Память задачи (`core/task_state.py`)
- **`TaskStateManager`** - отслеживание состояния задачи
- **`TaskStateConfig`** - конфигурация менеджера
- Извлечение целей, деталей и ограничений из диалога

### Уровень сервисов (`services/`)

#### MCP оркестратор (`services/mcp_orchestrator.py`)
- **`MCPOrchestrator`** - управление несколькими MCP серверами
- **`MCPToolCall`** - запись о вызове инструмента
- Поддержка динамического подключения серверов

#### RAG сервис (`services/rag_service.py`)
- **`RAGService`** - семантический поиск по документам
- **`RAGResult`** - результат поиска
- Использует FAISS для векторного поиска

### Уровень утилит (`utils/`)

#### Конфигурация (`utils/config.py`)
- **`Settings`** - централизованная конфигурация (Pydantic)
- Загрузка из `.env` файла
- Валидация настроек

#### Асинхронные утилиты (`utils/async_helpers.py`)
- **`BackgroundEventLoop`** - фоновый event loop в отдельном потоке
- Утилиты для работы с асинхронным кодом

#### База данных (`utils/database.py`)
- **`DatabaseConnection`** - управление подключениями к SQLite
- **`DatabaseModel`** - базовый класс для моделей БД
- Транзакции и пул подключений

## 🔄 Потоки данных

### Обработка сообщения пользователя

```
1. Пользователь вводит сообщение
   ↓
2. UI отправляет сообщение в MessageHandler
   ↓
3. MessageHandler передает сообщение агенту
   ↓
4. Агент определяет тип запроса (MCP/RAG/Knowledge)
   ↓
5. Если MCP → MCPOrchestrator вызывает инструменты
   ↓
6. Если RAG → RAGService выполняет поиск
   ↓
7. ConversationManager сохраняет сообщение пользователя
   ↓
8. TaskStateManager обновляет состояние задачи
   ↓
9. Summarizer проверяет необходимость суммаризации
   ↓
10. Агент формирует промпт с контекстом
   ↓
11. LLM (OpenRouter) генерирует ответ
   ↓
12. ConversationManager сохраняет ответ ассистента
   ↓
13. UI обновляет историю чата
```

### Переключение диалога

```
1. Пользователь выбирает диалог из списка
   ↓
2. ConversationHandler меняет активный диалог
   ↓
3. ConversationManager загружает историю сообщений
   ↓
4. UI очищает чат и загружает новую историю
   ↓
5. TaskStateManager загружает состояние задачи
   ↓
6. Summarizer загружает последнюю суммаризацию
```

```sql
CREATE TABLE conversations (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    title TEXT DEFAULT 'Новый диалог',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    active BOOLEAN DEFAULT 1
);
```

### Таблица `messages`
```sql
CREATE TABLE messages (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    conversation_id INTEGER NOT NULL,
    role TEXT NOT NULL CHECK (role IN ('user', 'assistant', 'system')),
    content TEXT NOT NULL,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    is_summary BOOLEAN DEFAULT 0,
    FOREIGN KEY (conversation_id) REFERENCES conversations(id)
);
```

### Таблица `task_states`
```sql
CREATE TABLE task_states (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    conversation_id INTEGER NOT NULL,
    clarified_details JSON DEFAULT '[]',
    constraints JSON DEFAULT '{}',
    goal TEXT DEFAULT '',
    last_summary TEXT DEFAULT '',
    message_count INTEGER DEFAULT 0,
    last_summarized_at_message INTEGER DEFAULT 0,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (conversation_id) REFERENCES conversations(id)
);
```

## 🎯 Принципы проектирования

### 1. Разделение ответственности
Каждый модуль отвечает за одну конкретную задачу:
- UI компоненты - только отображение
- Обработчики - только логика обработки событий
- Сервисы - только работа с внешними системами

### 2. Слабая связанность
Модули взаимодействуют через четко определенные интерфейсы:
- Конфигурация через Pydantic Settings
- Состояние через dataclasses
- События через callback функции

### 3. Расширяемость
Новые функции можно добавлять без изменения существующего кода:
- Новые MCP инструменты через конфигурацию
- Новые модели LLM через настройки
- Новые компоненты UI через наследование

### 4. Отказоустойчивость
- Фоновый event loop для асинхронных операций
- Graceful shutdown при остановке
- Резервные ответы при ошибках LLM

## 🔍 Алгоритмы и логика

### Выбор инструмента
```python
def _select_tool(message: str) -> ToolSelection:
    # Подсчет ключевых слов для MCP
    mcp_keywords = ["маршрут", "расстояние", "координат", "город", "расчет"]
    mcp_score = sum(1 for keyword in mcp_keywords if keyword in message_lower)
    
    # Подсчет ключевых слов для RAG
    rag_keywords = ["тариф", "правило", "документ", "постановление"]
    rag_score = sum(1 for keyword in rag_keywords if keyword in message_lower)
    
    # Определение выбора
    if mcp_score > 0 and rag_score > 0:
        return ToolSelection.HYBRID
    elif mcp_score > rag_score:
        return ToolSelection.MCP
    elif rag_score > 0:
        return ToolSelection.RAG
    else:
        return ToolSelection.KNOWLEDGE
```

### Автосуммаризация
```python
def should_summarize(conversation_id: int) -> bool:
    user_count = get_user_message_count(conversation_id)
    last_summarized = get_last_summarized_count(conversation_id)
    
    # Суммаризируем каждые 10 сообщений пользователя
    return user_count >= 10 and (user_count - last_summarized) >= 10
```
