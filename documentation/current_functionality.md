# Описание текущей функциональности проекта Logsit Agent

## Общая информация

**Logsit Agent** - это интеллектуальный ассистент логиста, который объединяет современные технологии для автоматизации логистических задач. Проект реализует гибридный подход, сочетающий LLM-агента с доступом к специализированным инструментам через MCP (Model Context Protocol) и RAG (Retrieval-Augmented Generation) систему для поиска информации в документах.

## Основные возможности

### 1. Планирование маршрутов между городами

**Функциональность:**
- Расчет оптимального маршрута между городами (задача коммивояжера)
- Геокодирование городов через Яндекс API
- Поддержка до 5 городов в одном маршруте
- Форматирование результатов в читаемый вид

**Техническая реализация:**
- MCP сервер `mcp_server.py` с инструментами:
  - `geocode_batch` - получение координат городов
  - `find_optimal_route` - решение задачи коммивояжера
  - `calculate_distance` - расчет расстояния между двумя городами
  - `format_route_summary` - форматирование результатов

### 2. Расчет стоимости доставки через ПЭК

**Функциональность:**
- Расчет стоимости доставки через API транспортной компании ПЭК
- Автоматическое определение ID городов ПЭК
- Защита от ошибок API с использованием фиксированного тарифа

**Техническая реализация:**
- MCP сервер `pecom_server.py` с инструментом:
  - `calculate_cost` - расчет стоимости доставки
- Интеграция с публичным API ПЭК для получения списка городов

### 3. Интеллектуальный поиск в документах перевозчиков

**Функциональность:**
- Семантический поиск по документам перевозчиков и нормативным актам
- Поддержка TXT и PDF файлов
- Автоматическая индексация документов с разбиением на чанки
- Две стратегии чанкинга: фиксированный размер и по предложениям

**Техническая реализация:**
- RAG система с компонентами:
  - `indexer.py` - индексатор документов
  - `rag_retriever.py` - поисковый модуль
- Использование FAISS для векторного поиска
- SQLite база для метаданных чанков

### 4. Веб-интерфейс для взаимодействия

**Функциональность:**
- Чат-интерфейс на основе Gradio
- Поддержка истории сообщений
- Выбор моделей LLM через OpenRouter
- Отладочная панель с логом вызовов инструментов
- Автоматическая инициализация компонентов

**Техническая реализация:**
- `app.py` - основной файл веб-интерфейса
- Интеграция с агентом `agent.py`
- Асинхронная обработка сообщений

## Архитектура системы

### Компоненты и их взаимодействие

```
Пользователь → Gradio UI → LLM Agent → Инструменты
                                      ↓
                    RAG System ↔ MCP Orchestrator
                                      ↓
                    Яндекс API ↔ ПЭК API ↔ Документы
```

### Основные компоненты

1. **LLM Agent (`agent.py`)** - центральный интеллектуальный компонент:
   - Интеллектуальная маршрутизация между RAG и MCP инструментами
   - Подключение к нескольким MCP серверам через оркестратор
   - Ограничение на количество городов (максимум 5)
   - Обработка ошибок и fallback стратегии

2. **MCP Orchestrator** - управление несколькими MCP серверами:
   - Подключение серверов геокодирования и расчета стоимости
   - Префиксная маршрутизация инструментов
   - Обработка ошибок подключения

3. **RAG System** - семантический поиск в документах:
   - Индексация документов с использованием sentence-transformers
   - FAISS для эффективного векторного поиска
   - SQLite для хранения метаданных

### Алгоритм обработки запроса

1. **Анализ типа запроса** - агент определяет стратегию обработки:
   - Информационные запросы → RAG поиск
   - Расчетные запросы → MCP инструменты

2. **RAG обработка** (для информационных запросов):
   - Поиск релевантных чанков в документах
   - Формирование контекста для LLM
   - Генерация ответа с указанием источников

3. **MCP обработка** (для расчетных запросов):
   - Последовательный вызов инструментов
   - Передача данных между инструментами
   - Форматирование финального ответа

## Технологический стек

### Основные технологии
- **Python 3.10+** - основной язык разработки
- **OpenRouter** - доступ к различным LLM моделям
- **MCP (Model Context Protocol)** - протокол для инструментов
- **FAISS** - библиотека для векторного поиска
- **Sentence Transformers** - генерация эмбеддингов
- **SQLite** - база данных для метаданных
- **Gradio** - веб-интерфейс

### API интеграции
- **Яндекс Карты API** - геокодирование городов
- **ПЭК API** - расчет стоимости доставки

### Модели LLM
Поддерживаются различные модели через OpenRouter:
- Бесплатные: Qwen 3.6 Plus Preview, Step-3.5 Flash, OpenRouter Free
- Платные: Xiaomi MiMo v2 Pro, MiniMax M2.5, DeepSeek v3.2

## Текущие ограничения

### Функциональные ограничения
- Максимум 5 городов в одном маршруте
- Фиксированный тариф ПЭК при недоступности API
- Ограниченный набор документов в RAG системе

### Технические ограничения
- Зависимость от внешних API (Яндекс Карты, ПЭК)
- Требуется API ключ OpenRouter для работы
- Ограничения по производительности на CPU
