# Расширенные функции Logsit Agent

## Обзор

Версия 2.0 Logsit Agent включает три ключевые новые функции:
1. **Множество диалогов с переключением** - поддержка нескольких параллельных диалогов
2. **Автоматическая суммаризация** - автоматическое создание кратких выжимок каждые 10 сообщений пользователя
3. **Task State (память задачи)** - отслеживание контекста диалога, уточнений и ограничений

## Архитектура

### Компоненты

1. **ConversationManager** (`conversation_manager.py`) - управление диалогами:
   - Создание, удаление, переключение диалогов
   - Хранение истории сообщений в SQLite базе данных
   - Статистика по диалогам

2. **Summarizer** (`summarizer.py`) - автоматическая суммаризация:
   - Триггер каждые 10 сообщений пользователя
   - Использование LLM для генерации лаконичных выжимок
   - Сохранение суммаризаций для контекста

3. **TaskStateManager** (`task_state.py`) - память задачи:
   - Отслеживание уточнений пользователя
   - Фиксация ограничений и терминов
   - Определение цели диалога
   - Интеграция с промптом LLM

4. **EnhancedRoutePlannerAgent** (`route_planner/enhanced_agent.py`) - обновленный агент:
   - Интеграция с новыми компонентами
   - Обновленный промпт с контекстом диалога
   - Автоматическая проверка триггеров суммаризации

5. **Enhanced UI** (`route_planner/enhanced_app.py`) - обновленный интерфейс:
   - Список диалогов с переключением
   - Информация о текущем диалоге
   - Управление диалогами (создание, удаление)

### База данных

```
conversations.db
├── conversations (диалоги)
│   ├── id INTEGER PRIMARY KEY
│   ├── title TEXT
│   ├── created_at DATETIME
│   ├── updated_at DATETIME
│   ├── active BOOLEAN
│   ├── message_count INTEGER
│   └── user_message_count INTEGER
├── messages (сообщения)
│   ├── id INTEGER PRIMARY KEY
│   ├── conversation_id INTEGER REFERENCES conversations(id)
│   ├── role TEXT (user/assistant/system)
│   ├── content TEXT
│   ├── timestamp DATETIME
│   └── is_summary BOOLEAN
└── task_states (состояния задач)
    ├── id INTEGER PRIMARY KEY
    ├── conversation_id INTEGER REFERENCES conversations(id)
    ├── clarified_details JSON (уточнения)
    ├── constraints JSON (ограничения)
    ├── goal TEXT (цель)
    ├── last_summary TEXT (последняя суммаризация)
    ├── message_count INTEGER
    ├── last_summarized_at_message INTEGER
    └── updated_at DATETIME
```

### Основные функции интерфейса

1. **Создание нового диалога** - кнопка "➕ Новый диалог"
2. **Переключение между диалогами** - выпадающий список "Все диалоги"
3. **Просмотр информации о диалоге** - панель справа
4. **Удаление диалогов** - кнопка "❌ Удалить диалог"
5. **Очистка истории текущего диалога** - кнопка "🗑️ Очистить текущий диалог"

### Автоматическая суммаризация

Суммаризация происходит автоматически:
- После 10-го сообщения пользователя в диалоге
- Далее каждые 10 сообщений пользователя
- Суммаризация сохраняется как системное сообщение
- Используется для контекста в последующих ответах

### Task State (память задачи)

Система автоматически отслеживает:
- **Цель диалога** (например, "рассчитать маршрут между городами")
- **Уточненные детали** (города, вес, сроки, бюджет)
- **Ограничения** (максимальное количество городов, лимиты)
- **Термины** (определения логистических терминов)

## API модулей

### ConversationManager

```python
from conversation_manager import get_conversation_manager

# Получение менеджера
manager = get_conversation_manager()

# Создание диалога
conversation = manager.create_conversation("Мой диалог")

# Добавление сообщения
message = manager.add_message(conversation.id, "user", "Привет!")

# Получение всех диалогов
conversations = manager.get_all_conversations()

# Переключение активного диалога
manager.set_active_conversation(conversation.id)

# Получение сообщений диалога
messages = manager.get_conversation_messages(conversation.id)
```

### Summarizer

```python
from summarizer import get_summarizer
from conversation_manager import get_conversation_manager

manager = get_conversation_manager()
summarizer = get_summarizer(manager)

# Проверка необходимости суммаризации
should_summarize, count = manager.should_summarize(conversation.id)

# Автоматическая суммаризация
summary = await summarizer.check_and_summarize(conversation.id)

# Принудительная суммаризация
summary = await summarizer.force_summarize(conversation.id)

# Получение контекста с суммаризацией
context = summarizer.get_summary_context(conversation.id)
```

### TaskStateManager

```python
from task_state import get_task_state_manager
from conversation_manager import get_conversation_manager

manager = get_conversation_manager()
task_manager = get_task_state_manager(manager)

# Обновление состояния на основе сообщения
from conversation_manager import Message
message = Message(role="user", content="Нужно рассчитать маршрут")
state = await task_manager.update_from_new_message(conversation.id, message)

# Получение контекста для LLM
context = task_manager.get_context_for_llm(conversation.id)

# Сброс состояния
task_manager.reset_task_state(conversation.id)
```

## Конфигурация

### Настройки суммаризации

В `summarizer.py`:
```python
class SummarizerConfig:
    summarization_trigger_count: int = 10  # Каждые N сообщений
    summarization_model: str = "openrouter/free"  # Модель для суммаризации
    max_tokens: int = 300  # Максимальная длина суммаризации
```

### Настройки Task State

В `task_state.py`:
```python
# Паттерны для извлечения информации
detail_patterns = [
    (r'(?:город[а]?|маршрут)[\s\w]*:?\s*([А-Я][а-я]+(?:\s+[А-Я][а-я]+)*)', ...),
    (r'(?:вес|масса)[\s\w]*:?\s*(\d+)\s*(?:кг|килограмм)', ...),
    (r'(?:срок|время|доставк[аи])[\s\w]*:?\s*(\d+)\s*(?:час|день|сутк)', ...),
]

# Термины логистики
logistics_terms = {
    'фрахтователь': 'Заказчик перевозки, нанимающий перевозчика',
    'фрахтовщик': 'Перевозчик, оказывающий услуги по перевозке',
    # ...
}
```

## Ограничения и известные проблемы

### Текущие ограничения

1. **Производительность SQLite** - при большом количестве диалогов (>1000) может замедлиться
2. **Качество суммаризации** - зависит от качества LLM модели
3. **Извлечение состояния** - rule-based подход может пропускать сложные конструкции

### Планируемые улучшения

1. **Улучшение извлечения состояния** - использование более сложных NLP методов
2. **Кэширование суммаризаций** - для уменьшения количества вызовов LLM
3. **Экспорт диалогов** - возможность экспорта истории в форматы CSV/JSON
4. **Поиск по диалогам** - полнотекстовый поиск по истории сообщений
5. **Темы диалогов** - автоматическая классификация диалогов по темам

## Устранение неполадок

### Распространенные проблемы

1. **Ошибка "Value: 1 is not in the list of choices: []"**
   - Причина: Проблема с инициализацией dropdown в UI
   - Решение: Перезагрузить страницу или перезапустить приложение

2. **Суммаризация не работает**
   - Проверить наличие API ключа OpenRouter
   - Проверить количество сообщений пользователя (должно быть ≥10)
   - Проверить логи на наличие ошибок LLM

3. **Task State не извлекает информацию**
   - Проверить паттерны извлечения в `task_state.py`
   - Убедиться, что сообщения сохраняются в БД

4. **Медленная работа с большим количеством диалогов**
   - Оптимизировать запросы к БД
   - Рассмотреть переход на более производительную БД
