# Расширенные функции Logsit Agent

## Добавленные возможности

### 1. Поддержка нескольких диалогов с переключением

**Описание:**
- Пользователь может вести несколько независимых диалогов одновременно
- Возможность переключения между диалогами без потери контекста
- Автоматическое сохранение истории в SQLite базе данных

**Реализация:**
- `conversation_manager.py` - управление диалогами и БД
- SQLite база `conversations.db` с таблицами:
  - `conversations` - информация о диалогах
  - `messages` - история сообщений  
  - `task_states` - состояния задач

**Использование:**
```python
from conversation_manager import get_conversation_manager

manager = get_conversation_manager()
# Создание нового диалога
conv = manager.create_conversation("Расчет маршрута")
# Добавление сообщения
manager.add_message(conv.id, "user", "Рассчитай маршрут...")
# Получение истории
messages = manager.get_conversation_messages(conv.id)
```

### 2. Автоматическая суммаризация каждые 10 сообщений

**Описание:**
- Автоматическое создание кратких сводок диалога
- Триггер каждые 10 сообщений пользователя
- Использование LLM для генерации лаконичных суммаризаций
- Сохранение сводок для контекста в последующих ответах

**Реализация:**
- `summarizer.py` - модуль суммаризации
- Настраиваемые параметры (количество сообщений, модель LLM)
- Интеграция с ConversationManager

**Конфигурация:**
```python
from summarizer import Summarizer, SummarizerConfig

config = SummarizerConfig(
    summarization_trigger_count=10,  # Каждые 10 сообщений
    summarization_model="openrouter/free",
    max_tokens=300
)
summarizer = Summarizer(manager, config)
```

### 3. Task State (память задачи)

**Описание:**
- Отслеживание состояния задачи в диалоге
- Автоматическое извлечение уточнений, ограничений и целей
- Поддержка контекста для улучшения ответов LLM

**Структура состояния:**
```python
{
    "goal": "Рассчитать оптимальный маршрут между Москвой и СПб",
    "clarified_details": [
        "Вес груза: 150 кг",
        "Срок доставки: 2 дня",
        "Бюджет: до 20,000 руб"
    ],
    "constraints": {
        "max_cities": 5,
        "max_weight": 1000,
        "time_limit": "48 часов"
    },
    "terms": {
        "фрахтователь": "Заказчик перевозки",
        "фрахтовщик": "Перевозчик"
    }
}
```

**Реализация:**
- `task_state.py` - модуль состояния задачи
- Rule-based извлечение информации из сообщений
- Опциональная интеграция с LLM для улучшения анализа

## Архитектурные изменения

### Обновленные компоненты

1. **EnhancedRoutePlannerAgent** (`enhanced_agent.py`)
   - Интеграция с ConversationManager, Summarizer и TaskStateManager
   - Поддержка контекста диалога в промптах LLM
   - Автоматическая проверка триггеров суммаризации

2. **Enhanced UI** (`enhanced_app.py`)
   - Интерфейс переключения диалогов
   - Отображение информации о текущем диалоге
   - Управление состоянием задачи

3. **База данных диалогов**
   - Отдельная от RAG системы
   - Схема с поддержкой отношений
   - Индексы для быстрого поиска

### Поток данных

```
Пользователь → Enhanced UI → EnhancedRoutePlannerAgent
                    ↓                    ↓
           ConversationManager      LLM + MCP/RAG
                    ↓                    ↓
              Summarizer            TaskStateManager
                    ↓                    ↓
              SQLite DB              Context для LLM
```

## Ограничения и будущие улучшения

### Текущие ограничения:

1. **Производительность:** SQLite может замедляться при большом количестве диалогов
2. **Качество суммаризации:** Зависит от выбранной модели LLM
3. **Извлечение информации:** Rule-based подход может пропускать сложные контексты

### Планируемые улучшения:

1. **Веб-интерфейс:** Улучшенное управление диалогами
2. **Экспорт данных:** Возможность экспорта истории диалогов
3. **Аналитика:** Статистика использования и эффективности
4. **Интеграции:** API для внешних систем

## Миграция с предыдущей версии

Расширенная версия является надстройкой над существующей архитектурой:
- Все существующие функции сохраняются
- Старый код остается работоспособным
- Возможность параллельного использования двух версий

Для миграции достаточно:
1. Установить новые зависимости (если требуются)
2. Запустить `enhanced_app.py` вместо `app.py`
3. Данные из старой версии не переносятся автоматически
