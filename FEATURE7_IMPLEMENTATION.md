# FEATURE 7: Интеграция локальной модели Ollama

## Реализованные изменения

### 1. Абстрактный слой LLMClient (`llm_client.py`)
- **LLMClient** - абстрактный класс с методами `chat_completion()` и `supports_tools()`
- **OpenRouterClient** - клиент для работы с OpenRouter (поддерживает function calling)
- **OllamaClient** - клиент для работы с локальной моделью через Ollama (не поддерживает tools)
- Фабрика `create_llm_client()` для создания клиента в зависимости от режима

### 2. Обновления EnhancedRoutePlannerAgent (`route_planner/enhanced_agent.py`)
- Добавлен параметр `use_local` в конструктор
- Динамическая инициализация клиента LLM через `create_llm_client()`
- Метод `_get_system_prompt()` возвращает разные промпты для локальной/OpenRouter моделей
- **BASE_SYSTEM_PROMPT_MCP** - промпт с MCP инструментами для OpenRouter
- **BASE_SYSTEM_PROMPT_LOCAL** - промпт без MCP для локальной модели
- Обновлен метод `_process_with_mcp()` для обработки локальной модели (возвращает предупреждение)

### 3. Обновления UI (`route_planner/enhanced_app.py`)
- **use_local_checkbox** - чекбокс для выбора локальной модели
- **local_model_info** - информационное поле о режиме локальной модели
- Функция `update_model_ui()` - обновляет UI в зависимости от выбора модели
- Функция `initialize_app()` - инициализирует приложение с выбранными параметрами
- Функция `get_mcp_status()` - показывает статус с учетом типа модели

### 4. Обработка ограничений локальной модели
- Локальная модель не поддерживает function calling (tools)
- MCP инструменты недоступны при использовании Ollama
- RAG работает с обоими типами моделей
- При запросах к MCP выводится предупреждение

## Использование

### Через UI:
1. В боковой панели настройте модель:
   - Чекбокс "Использовать локальную модель Ollama"
   - Выбор модели OpenRouter (если не выбрана локальная)
   - Кнопка "Инициализировать модель"

### Программно:
```python
from route_planner.enhanced_agent import EnhancedRoutePlannerAgent

# Локальная модель
agent_local = EnhancedRoutePlannerAgent(
    model="llama3.2:3b",
    use_local=True
)

# OpenRouter модель
agent_cloud = EnhancedRoutePlannerAgent(
    model="openrouter/auto",
    use_local=False
)
```

## Конфигурация

### Переменные окружения (опционально):
```
# Для локальной модели
OLLAMA_URL=http://localhost:11434
OLLAMA_MODEL=llama3.2:3b

# Для OpenRouter
OPENROUTER_API_KEY=your_api_key_here
```

## Ограничения локальной модели
1. ❌ Не поддерживает function calling (MCP инструменты)
2. ❌ Не может выполнять расчеты маршрутов и стоимости
3. ✅ Поддерживает RAG для поиска в документах
4. ✅ Может отвечать на общие вопросы

## Архитектурные преимущества
1. Единый интерфейс LLMClient для всех провайдеров
2. Легкое добавление новых провайдеров LLM
3. Прозрачное переключение между локальной и облачной моделями
4. Сохранение обратной совместимости с существующим кодом
