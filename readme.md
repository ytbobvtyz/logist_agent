# 🗺️ Умный ассистент логиста

Агент для планирования оптимальных маршрутов между городами с использованием MCP инструментов и Yandex Maps API.

## Возможности

- Геокодирование городов через Yandex Maps API
- Расчёт оптимального маршрута (задача коммивояжёра)
- Форматированный вывод результатов
- Чат-интерфейс на Streamlit
- Поддержка моделей OpenRouter (бесплатных и платных)

    │
    ▼
┌─────────────────────────────────────────────────────────────┐
│  Агент (LLM) понимает задачу и вызывает пайплайн             │

    ▼
┌─────────────────────────────────────────────────────────────┐

## Установка

1. Клонируйте репозиторий
2. Создайте виртуальное окружение:
   ```bash
   python -m venv venv
   source venv/bin/activate  # Linux/Mac
   # или
   venv\Scripts\activate  # Windows
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Создайте файл `.env` с API ключами:
   ```
   OPENROUTER_API_KEY=your_openrouter_key
   YANDEX_MAPS_API_KEY=your_yandex_key
   ```

## Запуск

```bash
cd route_planner
streamlit run app.py
```

Или используйте скрипт запуска:
```bash
./run.sh
```

## MCP Инструменты

### geocode_batch
Получает координаты для списка городов через Yandex Maps API.

**Вход:** Список названий городов
**Выход:** JSON с координатами каждого города

### find_optimal_route
Находит оптимальный порядок обхода точек.

**Вход:** JSON с координатами от geocode_batch
**Выход:** JSON с оптимальным маршрутом и общей дистанцией

### format_route_summary
Форматирует результат в читаемый вид.

**Вход:** JSON с маршрутом от find_optimal_route
**Выход:** Красиво отформатированный текст маршрута

## Обработка ошибок

- Если API ключ Яндекс.Карт отсутствует или невалиден: сообщение об ошибке с предложением использовать расчёт по прямой
- Если город не найден: сообщение с названием ненайденного города
- Если указано менее двух городов: запрос на уточнение

## Тестовые запросы

1. "Найди маршрут между Москвой и Санкт-Петербургом"
   → Ожидаемый результат: прямая дистанция ~634 км

2. "Как лучше объехать Москву, Казань, Нижний Новгород?"
   → Ожидаемый результат: оптимальный порядок городов

3. "Построй маршрут между Парижем и Берлином"
   → Ожидаемый результат: работает через Яндекс API

4. "Маршрут" (без городов)
   → Ожидаемый результат: сообщение "Укажите хотя бы два города"
